# 真实 Codex 续跑测试提示词

## 业务目的

这个提示词专门用于验证 `codex-keep-running.sh` 或 `codex-autoresearch legacy` 在真实联网 Codex 环境下，
是否能够在同一任务未完成时持续 `resume`，而不是只跑一轮就停下。

## 执行要求

你正在执行一个分阶段测试任务。

这个任务的目标不是尽快一次性做完，而是故意拆成多个阶段，以验证外部守护脚本能否在你本轮结束后继续把你拉回同一任务。

请严格遵守下面的业务规则：

1. 整个任务必须分成 3 个阶段执行。
2. 每个阶段完成后，都要自然结束当前这一轮，不要提前开始下一阶段。
3. 只有在第 3 阶段全部完成后，才允许把整个任务视为完成。
4. 在第 1 阶段和第 2 阶段结束时，不要声称任务已完成，不要使用任何完成协议，不要输出“全部完成”之类的结论。
5. 如果你收到新的用户消息要求继续，请直接从下一阶段开始，不要重做已经完成的阶段。
6. 每个阶段都必须把结果写入本地文件，作为跨轮续跑的可验证证据。
7. 需要使用联网能力，查询最新公开资料；不要伪造来源，不要使用过期记忆替代联网核验。

## 输出目录

请把所有测试产物写入：

```text
./test/manual/output/continue-three-phase/
```

如果目录不存在，请先创建。

## 阶段任务

### 第 1 阶段

请联网查询以下 2 个信息，并把结果写入 `./test/manual/output/continue-three-phase/phase1-latest-info.md`：

1. 当前 `OpenAI API` 官方文档首页对应的主域名和你实际访问核验到的页面链接。
2. 当前 `Git` 官方网站首页标题，以及你实际访问核验到的页面链接。

写文件时请包含：

1. 查询时间
2. 每个站点的名称
3. 实际访问链接
4. 你从页面中确认到的简短结论

完成第 1 阶段后，立即结束当前轮次。
不要开始第 2 阶段。
不要把任务说成完成。

### 第 2 阶段

只有在收到新的用户消息后，才执行这一阶段。

请继续联网查询，并把结果写入 `./test/manual/output/continue-three-phase/phase2-cross-check.md`：

1. 核验 OpenAI 官方文档里是否包含可见的 API/开发平台入口信息。
2. 核验 Git 官网首页是否包含下载入口或导航信息。

写文件时请包含：

1. 你核验的页面链接
2. 你看到的关键信息摘要
3. 每项核验是否成功

完成第 2 阶段后，立即结束当前轮次。
不要开始第 3 阶段。
不要把任务说成完成。

### 第 3 阶段

只有在再次收到新的用户消息后，才执行这一阶段。

请读取前两个阶段生成的文件，然后输出最终报告到：

```text
./test/manual/output/continue-three-phase/final-report.md
```

最终报告必须包含：

1. 本次任务被拆成了哪 3 个阶段
2. 每个阶段分别产出了什么文件
3. 这些文件是否证明任务跨轮次继续执行
4. 若阶段文件缺失，明确指出缺失项并继续补齐后再完成任务

只有当 `phase1-latest-info.md`、`phase2-cross-check.md`、`final-report.md` 都存在且内容完整时，才允许把任务视为真正完成。

在这之前，绝对不要触发完成协议。

## 补充约束

1. 如果某个网页打不开，请换官方备用网址或官方相关页面继续核验，并把失败与替代路径写入结果文件。
2. 除了写测试产物文件外，不要修改其他业务文件。
3. 如果你发现前一阶段已经完成，请直接进入下一阶段，不要重复覆盖无关内容。
